# Prompt Experiments

This directory contains records of prompt experiments, A/B tests, and performance evaluations.

---

## Purpose

Experiment records help us:

- Compare different prompt approaches
- Track what works and what doesn't
- Share learnings with the team
- Improve prompts systematically

---

## Experiment Record Template

Create a new file for each experiment:

```markdown
# Experiment: {experiment_name}

**Date:** YYYY-MM-DD
**Author:** Name
**Status:** [Planned | In Progress | Complete]

## Objective
What are you trying to learn or compare?

## Hypothesis
What do you expect to happen?

## Methodology
How will you run this experiment?

### Prompts Tested

#### Prompt A
```
[Prompt version A]
```

#### Prompt B
```
[Prompt version B]
```

## Results

| Metric | Prompt A | Prompt B |
|--------|----------|----------|
| Accuracy | % | % |
| Consistency | Rating | Rating |
| Token Usage | Count | Count |

## Observations
- Observation 1
- Observation 2

## Conclusion
What did you learn?

## Recommendations
How should prompts be updated based on this?
```

## Best Practices

1. **Define clear metrics** - How will you measure success?
2. **Control variables** - Change one thing at a time
3. **Document thoroughly** - Future you will thank you
4. **Be objective** - Report what you observe, not what you hope
5. **Iterate** - Use learnings to design better experiments

---

## Metrics to Track

### Quality Metrics
- Accuracy of outputs
- Relevance to request
- Completeness of response

### Performance Metrics
- Response time
- Token usage
- Cost per request

### Consistency Metrics
- Same input, same output reliability
- Format adherence
- Style consistency
