# Meeting Notes

This directory contains records of team meetings, discussions, and decisions.

## Naming Convention

```
YYYY-MM-DD-meeting-type.md
```

Examples:
- `2026-04-13-sprint-planning.md`
- `2026-04-15-architecture-review.md`
- `2026-04-20-retrospective.md`

## Meeting Note Template

```markdown
# Meeting: [Title]

**Date:** YYYY-MM-DD
**Time:** HH:MM - HH:MM
**Attendees:** List of attendees
**Facilitator:** Name
**Note Taker:** Name

## Agenda

1. Item 1
2. Item 2
3. Item 3

## Discussion Notes

### Topic 1
- Key point
- Discussion points
- Questions raised

### Topic 2
- Key point
- Discussion points

## Decisions Made

| Decision | Rationale | Owner |
|----------|-----------|-------|
| Decision 1 | Why | Who |

## Action Items

- [ ] Action 1 - @assignee - Due: date
- [ ] Action 2 - @assignee - Due: date

## Next Meeting

Date: YYYY-MM-DD
Agenda items for next meeting:
- Item 1
- Item 2
```

## Tips for Good Meeting Notes

1. **Record, don't interpret** - Capture what was said, not what you think was meant
2. **Note decisions clearly** - Make action items and decisions prominent
3. **Assign owners and due dates** - Action items need accountability
4. **Distribute promptly** - Share notes within 24 hours
5. **Keep them searchable** - Use consistent formatting and naming
