## Backlog Process

### Adding Items

1. Create a GitHub issue first
2. Discuss and refine
3. Add to backlog with priority and points
4. Link to issue

### Moving to Sprint

1. During sprint planning, select items
2. Move from backlog to sprint document
3. Update status in issue

### Refinement

- Weekly backlog refinement sessions
- Reprioritize as needed
- Remove stale items
