# Reference Documentation

Technical reference for APIs, configurations, and specifications.

---

## What is Reference Documentation?

Reference documentation is **information-oriented** - it describes the technical details of the system. It's designed for looking up specific information, not for learning.

## Reference Guidelines

### Characteristics of Good Reference Docs

1. **Complete** - Cover all options/parameters
2. **Accurate** - Match the actual implementation
3. **Consistent** - Same format throughout
4. **Searchable** - Easy to find what you need
5. **Examples** - Include usage examples

### Structure

For each item, document:

- Name/identifier
- Description
- Parameters/options
- Return values
- Examples
- Error cases
- Related items
